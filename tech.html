System Architecture

From skin to feedback, in six steps.

The Smart Compression Sleeve streams synchronized IMU, EMG, and PPG sensor data through a real-time Python processing pipeline for filtering, feature extraction, machine learning classification, and live rehabilitation feedback.

Hardware

Adafruit nRF52840 · Dual ICM-20948 IMUs · DFRobot EMG · DFRobot PPG · TCA9548A mux

Firmware

Arduino C++ · I²C · Analog read · USB Serial @ 115200 baud

Bridge & ML

Python · pyserial · pandas · scikit-learn · websockets

App & Cloud

Flutter · Dart · Supabase

Engineering Specifications

Optimized for real-time monitoring.

System parameters were selected to balance signal quality, real-time responsiveness, and stable multi-sensor streaming.

Core Technical Specifications

Parameter Selected Value Engineering Purpose
EMG Sampling Rate 1000 Hz Captures rapid muscle activation dynamics.
IMU Sampling Rate ~60 Hz Supports real-time upper-limb motion tracking.
PPG Update Rate ~1 Hz Appropriate for slower heart-rate variation.
Serial Baud Rate 115200 baud Stable simultaneous multi-sensor streaming.
ML Classifier Random Forest Robust multi-class exercise classification.

Signal Processing & Filters

Signal Applied Filtering Purpose
EMG 20–250 Hz bandpass + envelope smoothing Removes noise and extracts activation amplitude.
IMU Elbow Angle Complementary filter + 4 Hz low-pass Stabilizes elbow angle estimation and suppresses jitter.
PPG / Heart Rate Low-pass EMA smoothing Reduces pulse noise and smooths heart-rate signal.

Latency Validation

Signal End-to-End Latency Notes
IMU 2–50 ms Suitable for real-time motion tracking.
EMG 17–65 ms Includes analog filtering delay.
PPG 0.5–1 s Expected optical pulse sensing delay.
